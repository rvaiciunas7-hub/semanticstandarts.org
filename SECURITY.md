# Security Policy

## Supported Versions
This project follows a stable, modular, and institutionally controlled release model.  
Only explicitly designated versions receive security-related attention.

| Version | Supported |
|--------|-----------|
| Current | ✔ |
| Legacy  | ✖ |
| Pre‑release | ✖ |

## Reporting a Vulnerability
If you believe you have identified a security-related issue, please report it directly and privately.

To report a vulnerability:
- provide a clear description of the issue  
- include reproduction steps if applicable  
- avoid public disclosure before confirmation  

All reports must be submitted through a private communication channel.  
Public issue trackers must not be used for security disclosures.

## Security Principles
This project maintains:
- minimal and controlled attack surface  
- non‑derivative code and documentation  
- strict authorship and integrity constraints  
- stable, predictable versioning  

## Response Process
Upon receiving a report:
1. The issue will be reviewed.  
2. If confirmed, a fix or mitigation will be prepared.  
3. A coordinated disclosure timeline may be established.  

Security handling follows the institutional integrity rules defined by SSA Standards.
